# Pricing Models

## 1. Purpose

This document defines possible pricing models for AI QA Command Center.

The goal is to support honest product packaging without overselling the current framework.

---

## 2. Current Product Reality

AI QA Command Center is currently a repository-based AI-assisted QA operating framework.

It is strong as:

- Demo-ready framework
- Productized QA service foundation
- Internal QA operating kit
- AI-assisted QA documentation system
- Future dashboard / automation / integration base

It is not yet:

- A complete SaaS product
- A live dashboard application
- A fully autonomous QA execution platform
- A live Jira / Slack / Figma integration platform
- A replacement for QA engineers

Pricing must reflect this reality.

---

## 3. Pricing Principles

Use these principles:

| Principle | Meaning |
|---|---|
| Do not oversell | Do not price or describe it as a finished autonomous SaaS |
| Sell the current value | Sell structure, templates, demos, validation, and workflow |
| Separate service from software | Consulting/setup can be priced separately from repo access |
| Keep human review explicit | The framework supports humans; it does not replace them |
| Use staged pricing | Start with service/productized packages before full SaaS pricing |
| Keep execution out unless implemented | Do not sell automatic test execution until it exists |

---

## 4. Possible Pricing Models

| Model | Description | Best For | Current Fit |
|---|---|---|---|
| One-Time Framework License | Customer buys access to the repository/framework package | Agencies, internal QA teams | Medium |
| Productized Setup Service | You configure and adapt the framework for a customer | Startups, SMEs, agencies | High |
| Monthly Support Retainer | Ongoing improvements, review, and QA artifact support | Growing teams | High |
| Training Workshop | Teach customer teams how to use the framework | QA teams, agencies | High |
| Custom Implementation | Customize agents, templates, workflows, and demos | Larger companies | High |
| SaaS Subscription | Hosted dashboard/product subscription | Future product | Low today |
| Integration Add-On | Jira/Slack/GitHub/Figma/Firebase integration package | Future product | Low today |
| Automation Add-On | Generate or execute automated tests | Future product | Low today |

---

## 5. Recommended Current Pricing Direction

The strongest current pricing direction is not pure SaaS.

Recommended current model:

1. Productized setup service
2. Framework customization package
3. Training workshop
4. Monthly support / improvement retainer
5. Future SaaS after dashboard and execution features exist

Reason:

The repository is currently a strong framework foundation, not a finished app.

Selling it as setup + framework + training is more honest and more realistic.

---

## 6. Productized Setup Service Model

This is the best current model.

What is sold:

- Framework walkthrough
- Customer QA process analysis
- Custom feature intake templates
- Custom agent/prompt adjustments
- Custom demo feature pack for the customer's domain
- Validation setup
- GitHub Actions setup
- Onboarding session
- Human approval workflow definition

Possible pricing logic:

| Setup Size | Scope | Pricing Direction |
|---|---|---|
| Small Setup | Basic setup and walkthrough | Low fixed fee |
| Standard Setup | Custom templates and one customer demo | Medium fixed fee |
| Advanced Setup | Custom workflows, references, and team training | Higher fixed fee |
| Enterprise Setup | Multiple teams, custom governance, integration planning | Custom quote |

Do not include live integrations or execution unless specifically built.

---

## 7. Monthly Support Retainer Model

What is sold:

- Monthly framework improvements
- New demo feature packs
- Prompt refinements
- Output review support
- QA artifact generation support
- Roadmap planning
- Validation updates
- Training and office hours

Possible tiers:

| Retainer | Scope |
|---|---|
| Light Support | Monthly check-in and minor updates |
| Standard Support | Regular updates and QA output support |
| Advanced Support | Custom additions, team support, and roadmap work |

This is a strong model because the framework will need adaptation per customer.

---

## 8. Training Workshop Model

What is sold:

- How to use the framework
- How to write feature inputs
- How to inspect generated outputs
- How to run validation
- How to avoid overclaiming
- How to review outputs
- How to plan API and DB validation
- How to prepare release readiness drafts

Possible workshop formats:

| Format | Duration | Audience |
|---|---:|---|
| Intro Workshop | 1-2 hours | Managers, QA leads |
| Practical Workshop | Half day | QA team |
| Deep Implementation Workshop | 1-2 days | QA + Engineering team |

---

## 9. One-Time Framework License Model

This model can work, but it has risks.

Pros:

- Simple sale
- Easy to package
- No long-term obligation

Cons:

- Customer may not know how to use it
- Value may be misunderstood
- Harder to maintain relationship
- Easier to copy
- Less revenue than service/customization model

If used, include:

- Documentation
- Demo guide
- Getting started guide
- Limited onboarding session
- Clear license terms
- No promise of execution or integrations

---

## 10. SaaS Subscription Model

This is a future model.

It becomes realistic after adding:

- Real dashboard
- User/project management
- Generated output management
- Approval gates
- Integration configuration UI
- Workflow history
- Possibly automation generation
- Possibly execution evidence import

Do not sell the current repository as full SaaS.

Future SaaS pricing could include:

| Tier | Example Capability |
|---|---|
| Free / Demo | Limited demos and templates |
| Starter | Basic project and output workflow |
| Professional | Team workflows, integrations, reviews |
| Enterprise | Governance, security, SSO, custom agents |

This is roadmap, not current promise.

## 14. Pricing Risk

Do not charge like a finished SaaS while delivering a repository framework.

That creates trust damage.

Better positioning:

- Implementation package
- AI QA operating framework setup
- Productized QA enablement
- Custom QA workflow kit
- AI-assisted QA documentation system

---

## 15. Recommended Sales Motion

Recommended first sales motion:

1. Show demo.
2. Explain current limitations honestly.
3. Offer a paid setup package.
4. Customize the framework for customer domain.
5. Train the team.
6. Offer monthly support.
7. Later upsell dashboard/integration/automation features.

---

## 16. Final Recommendation

Best current pricing model:

Productized service + framework customization + training + optional retainer.

Avoid selling as autonomous SaaS until real dashboard, integrations, and execution features exist.
