# Acceptance Criteria Template

## Purpose

Use this template to define clear, testable acceptance criteria for a feature.

Acceptance criteria must be specific, observable, and testable.

## Good Acceptance Criteria Example

| ID | Criteria | Priority | User Role | Platform | Notes |
|---|---|---|---|---|---|
| AC-001 | Registered user can log in with valid email and password and is redirected to the dashboard. | P0 | Registered User | Web | Dashboard must show user full name. |

---

## Bad Acceptance Criteria Example

The login system works correctly.

Reason this is bad:

- Not specific
- Not observable
- Not testable
- No user role
- No expected result

---

## Validation Notes

For each acceptance criteria, consider:

- Happy path
- Edge cases
- Negative cases
- Permission cases
- API behavior
- DB impact
- UI states
- Regression risk
